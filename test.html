Methodology: 5‑Agent Architecture & Decision Workflow

1. Sensor Agent

Collects & preprocesses real‑time data: irradiance, temperature, voltage, current, dust (from inverter/API/sensors). Publishes to other agents via XMPP.

2. Diagnostic Agent

Rule‑based thresholds + Random Forest classification. Detects anomalies: dust, shading, inverter faults, degradation.

3. Predictive Agent

Runs LSTM (forecasting) + Autoencoder (unsupervised anomaly). Predicts failures 24‑48h ahead.

4. Decision Agent

Triggers cleaning, inspection, or replacement actions based on diagnostic/predictive scores.

5. Coordinator Agent

Aggregates signals, resolves conflicts, optimizes global maintenance schedule, escalates critical issues.
